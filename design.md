# 《余烬回廊》策划案

## 1. 世界观与玩法循环

玩家扮演一名在废弃炼金回廊中收集余烬能量的实验体。每次模拟代表一次短程探索：玩家带着一套初始加点与道具流派进入固定关卡，按怪物队列依次战斗，击败所有怪物则通关，玩家死亡则失败。模拟器的目标不是验证完整游戏乐趣，而是验证不同流派在相同关卡、相同规则下的胜率、耗时、伤害曲线与资源消耗差异。

本项目采用极简房间制肉鸽结构：玩家在多个封闭房间中清理怪物，清房后从随机道具里三选一，逐步形成 build。项目不追求完整商业化肉鸽体量，只保留一套足够小的角色、技能、道具、怪物和关卡，用来验证 `暴击流`、`持续输出流`、`召唤流` 三种流派的强弱。

核心循环：

1. 读取角色、技能、道具、怪物和关卡配置。
2. 外部策略 API 决定开局属性加点与携带道具。
3. 战斗开始，怪物按房间/波次队列依次登场。
4. 清理房间后进行道具三选一，强化当前 build。
5. 战斗结束后输出胜负、耗时、剩余 HP、技能次数、总伤害和伤害曲线。

## 2. 角色系统

### 2.1 可玩角色：余烬猎手

余烬猎手是一个中距离输出角色，基础机制围绕“爆发窗口”和“持续灼烧”两种方向设计。角色本身保持简单，强弱主要由属性加点、技能释放时机和道具组合体现。

| 属性 | 初始值 | 说明 |
| --- | ---: | --- |
| max_hp | 120 | 最大生命值 |
| attack | 12 | 普攻和技能的基础伤害来源 |
| defense | 3 | 每次受到伤害时的固定减伤 |
| crit_rate | 0.10 | 暴击率 |
| crit_damage | 1.8 | 暴击伤害倍率 |
| speed | 10 | 回合制先攻与实时模式行动频率 |
| burn_power | 4 | 灼烧每跳基础伤害 |
| summon_power | 6 | 召唤物的基础攻击力 |
| energy_regen | 1 | 每回合或每秒恢复的能量 |

### 2.2 属性加点机制

开局提供 6 点属性点，由策略 API 分配。每点只能投入一个属性，允许集中加点，用于放大不同流派差异。

| 加点项 | 每点效果 | 适合流派 |
| --- | --- | --- |
| attack | attack +2 | 暴击流、通用输出 |
| crit_rate | crit_rate +0.05 | 暴击流 |
| speed | speed +1.5 | 持续输出流、技能循环 |
| max_hp | max_hp +10 | 生存流或保底策略 |
| burn_power | burn_power +1.5 | 灼烧持续输出流 |
| summon_power | summon_power +2 | 召唤流 |
| defense | defense +1 | 抗压策略 |

加点本身不写死在战斗逻辑中，必须由策略实现返回。例如暴击流倾向 `attack + crit_rate`，灼烧流倾向 `speed + burn_power`，召唤流倾向 `summon_power + max_hp`。

## 3. 技能系统

技能消耗能量，拥有冷却时间。释放时机影响强弱：例如先用破甲技能再打爆发，在敌人血量较高时尽早叠灼烧，或提前召唤单位让它们多攻击几轮。

### 3.1 普攻：余烬短击

- 类型：默认动作
- 冷却：无
- 消耗：0
- 效果：造成 `attack` 点物理伤害，可暴击。
- 设计目的：当技能冷却或能量不足时的保底动作。

### 3.2 技能一：裂芯斩

- 冷却：3 回合或 3 秒
- 消耗：2 能量
- 效果：造成 `attack * 2.2` 点物理伤害，可暴击。
- 附加：如果目标身上有破甲，额外造成 25% 伤害。
- 设计目的：暴击流主力爆发技能，适合在暴击率较高或破甲后释放。

### 3.3 技能二：燃烬印记

- 冷却：4 回合或 4 秒
- 消耗：2 能量
- 效果：造成 `attack * 0.8` 点火焰伤害，并施加 4 层灼烧。
- 灼烧：每层每回合或每秒造成 `burn_power` 点伤害，持续 4 次结算。
- 设计目的：持续输出流核心技能，越早释放总收益越高。

### 3.4 技能三：碎甲脉冲

- 冷却：5 回合或 5 秒
- 消耗：3 能量
- 效果：造成 `attack * 1.0` 点伤害，施加 2 层破甲。
- 破甲：每层使目标受到的直接伤害提高 12%，持续 3 回合或 3 秒。
- 设计目的：为爆发技能创造窗口，释放顺序会显著影响结果。

### 3.5 技能四：余烬仆从

- 冷却：6 回合或 6 秒
- 消耗：3 能量
- 效果：召唤 1 个余烬仆从，持续 5 回合或 8 秒。
- 仆从行为：每次行动攻击当前目标，造成 `summon_power` 点火焰伤害。
- 限制：默认最多同时存在 2 个余烬仆从。
- 设计目的：召唤流核心技能，越早释放越能摊薄冷却收益；如果目标即将死亡时释放，则召唤持续时间会被浪费。

### 3.6 技能释放时机的验证点

- 暴击流若先释放 `碎甲脉冲` 再释放 `裂芯斩`，理论上单次爆发更高。
- 灼烧流若尽早释放 `燃烬印记`，理论上总伤害曲线更平滑，面对高 HP 敌人收益更高。
- 召唤流若在战斗前半段尽早释放 `余烬仆从`，理论上总输出更高；如果召唤过晚，收益会显著下降。
- 如果策略在低血量怪物身上浪费高冷却技能，耗时和胜率可能变差。

## 4. 道具系统

每次模拟开始由策略 API 选择 2 件道具。道具不是单纯加数值，而是让流派组合产生质变。

### 4.1 棱镜护符

- 效果：暴击率 +10%。
- 组合潜力：与 `裂芯斩`、`锋刃齿轮` 叠加后形成高爆发流。

### 4.2 锋刃齿轮

- 效果：暴击时额外造成 `attack * 0.6` 点追加伤害。
- 组合潜力：单独收益不稳定，配合高暴击率后收益明显上升。

### 4.3 余火沙漏

- 效果：灼烧持续次数 +1，灼烧伤害 +15%。
- 组合潜力：与 `燃烬印记` 和高 `burn_power` 加点配合，提升长线输出。

### 4.4 裂变火种

- 效果：目标身上灼烧层数达到 4 层时，立即触发一次小爆炸，造成 `burn_power * 3` 点伤害。
- 限制：每个目标每 3 秒或 3 回合最多触发一次。
- 组合潜力：与 `余火沙漏` 组合后，持续伤害变成“持续 + 周期爆发”的混合流派。

### 4.5 灰烬铃

- 效果：余烬仆从最大同时存在数量 +1。
- 组合潜力：让召唤流从“单个补伤害”变成“稳定铺场”。

### 4.6 共鸣核心

- 效果：每当余烬仆从造成伤害时，玩家恢复 0.3 能量。
- 组合潜力：与 `灰烬铃` 叠加后，召唤物越多，玩家技能循环越快，形成召唤流的质变点。

### 4.7 余烬急救瓶

- 类型：主动道具，1 次使用次数。
- 效果：策略返回 `use_item` 动作后，立即恢复固定 HP 与一部分最大 HP。
- 组合潜力：给长线流派提供容错，也用于验证策略 API 是否能真正控制“何时使用道具”，而不是只在开局选择被动装备。

## 5. 怪物系统

怪物 AI 保持简单，但种类上覆盖追击、快速追击、炮台、远程法师、慢速高血怪和精英守卫，保证房间体验有基础变化。

### 5.1 灰壳爬行者

| 属性 | 值 |
| --- | ---: |
| hp | 55 |
| attack | 8 |
| defense | 1 |
| speed | 8 |

行为：

- 每次行动使用普通攻击。
- 没有技能。
- 用作基础小怪，检验流派清怪效率。

### 5.2 炉心守卫

| 属性 | 值 |
| --- | ---: |
| hp | 150 |
| attack | 14 |
| defense | 4 |
| speed | 6 |

行为：

- 每次行动使用普通攻击。
- 每第 3 次行动进入 `guarded` 状态，受到的下一次直接伤害降低 30%。
- 灼烧伤害不受 `guarded` 减免影响。

设计目的：

- 高 HP 与防御用于测试持续输出流。
- `guarded` 会削弱单次爆发，使暴击流需要更好的释放时机。

### 5.3 其他怪物

- 火花蝠：低血高移速，压迫玩家走位。
- 骨火炮台：固定在原地，以远程攻击制造安全区压力。
- 焦油史莱姆：血量较高、移速较慢，用于测试持续输出收益。
- 余烬法师：中等血量远程怪，要求玩家优先处理威胁。

## 6. 关卡结构

### 6.1 房间结构

当前配置包含 6 个连续房间，用于形成类似《以撒》的短局流程。

怪物队列：

| 房间 | 核心怪物 | 设计目的 |
| --- | --- | --- |
| 回廊入口 | 灰壳爬行者、火花蝠 | 基础移动与射击教学 |
| 灰烬地窖 | 焦油史莱姆、骨火炮台 | 测试持续输出和站位 |
| 余烬书库 | 余烬法师、火花蝠 | 测试远程威胁处理 |
| 裂隙宝库 | 骨火炮台、焦油史莱姆 | 测试阵地压力和厚血敌人 |
| 火花巢穴 | 火花蝠、余烬法师 | 测试高速敌群和优先级判断 |
| 炉心大厅 | 灰壳爬行者、炉心守卫 | 最终强度验证 |

胜利条件：

- 击败所有怪物。

失败条件：

- 玩家 HP 降为 0。
- 超过最大模拟时长：80 回合或 120 秒。

## 7. 流派假设

### 7.1 流派 A：裂芯暴击流

核心思路：

- 加点集中在 `attack` 和 `crit_rate`。
- 道具选择 `棱镜护符` + `锋刃齿轮`。
- 战斗策略优先使用 `碎甲脉冲`，再使用 `裂芯斩` 打爆发。

预期优势：

- 面对低 HP 小怪时清理速度快。
- 如果关键技能暴击，伤害曲线会出现明显尖峰。
- 单场耗时可能更短。

预期劣势：

- 伤害波动大，批量模拟胜率可能不稳定。
- 面对 `炉心守卫` 的 `guarded` 状态时，如果释放时机差，爆发会被浪费。

验证命题：

- 暴击流是否能用更短平均耗时通关？
- 暴击流的胜率是否会因为随机性低于灼烧流？

### 7.2 流派 B：余火灼烧流

核心思路：

- 加点集中在 `speed` 和 `burn_power`。
- 道具选择 `余火沙漏` + `裂变火种`。
- 战斗策略优先在每个新目标身上尽早释放 `燃烬印记`，之后用普攻和技能补伤害。

预期优势：

- 伤害稳定，批量模拟结果方差较低。
- 面对高 HP、带防御或减伤的敌人时更有优势。
- 技能释放越早，总伤害收益越高。

预期劣势：

- 前几秒爆发不足，小怪清理速度可能慢。
- 如果敌人很快死亡，灼烧剩余持续时间会浪费。

验证命题：

- 灼烧流是否拥有更高胜率和更稳定的剩余 HP？
- 灼烧流是否在最终敌人阶段贡献更高总伤害？

### 7.3 流派 C：余烬召唤流

核心思路：

- 加点集中在 `summon_power` 和 `max_hp`。
- 道具选择 `灰烬铃` + `共鸣核心`。
- 战斗策略优先在每个高血量目标出现后尽早释放 `余烬仆从`，通过召唤物提供稳定伤害，并用回能效果加快后续技能循环。

预期优势：

- 伤害来源分散，不完全依赖暴击随机性或单个技能窗口。
- 面对 `炉心守卫` 这类高 HP 敌人时，召唤物持续攻击的总收益较高。
- `共鸣核心` 可能让玩家更频繁释放技能，形成后期滚雪球。

预期劣势：

- 启动慢，短战斗中召唤物可能还没打出足够收益。
- 如果召唤技能释放太晚，持续时间浪费严重。
- 需要额外记录召唤物数量、持续时间和伤害来源，实现复杂度略高于另外两种流派。

验证命题：

- 召唤流是否在最终敌人阶段拥有最高总伤害？
- 召唤流是否能用更高剩余 HP 换取更长平均耗时？
- 召唤流的稳定性是否介于暴击流和灼烧流之间？

## 8. 设计边界

本策划案刻意保持范围很小：

- 只有 1 个角色、4 个主动技能、13 件道具、6 种怪物、6 个房间。
- 不设计复杂地图、寻路、碰撞体积或真实动作表现。
- 战斗只需要支持回合制和 tick 制两套时间推进模型。
- 怪物 AI 简化为固定行为，避免抢走策略 API 的重点。

这套设计的重点是让模拟器能回答一个清晰问题：在同一关卡中，`裂芯暴击流`、`余火灼烧流` 和 `余烬召唤流` 到底谁更强，强在哪里，弱在哪里。

## 9. 后续扩展方向

- 新角色：增加护盾流、召唤流或道具触发流角色。
- 新怪物：增加抗暴击、抗灼烧、召唤小怪等机制。
- 新道具：当前清房奖励已支持随机词条，词条会改变道具显示名并合并运行时效果，可在批量模拟中观察词条组合收益。
- 新策略：接入外部 HTTP 或 stdio 策略进程，用 Python/LLM 返回决策。
- Replay 展示：当前已导出战斗事件流，并支持在 Godot UI 中按事件顺序播放上次结果；后续可扩展为可拖动时间轴和事件筛选。
